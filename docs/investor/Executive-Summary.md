# 단지카(DanjiCar) — Executive Summary

**한 줄 정의:** 아파트 입주민 전용 공유·렌트 차량 플랫폼 — 예약·결제·스마트키·반납·정산을 하나의 앱으로 운영

---

## Problem

- 단지 내 2차 주차·공유 차량 운영은 전화·엑셀·수기 검수에 의존하기 쉽고, 입주민 UX와 매출·가동률 가시성이 떨어짐
- 시간 단위 카셰어링과 일·월 렌트가 혼재할 때 요금·환불·겹침 예약·반납 검수를 일관되게 관리하기 어려움

## Solution

**단지카** — Flutter 앱 + Supabase 백엔드 + 토스페이먼츠

| 계층 | 구성 |
|------|------|
| 입주민 | 4탭 앱(홈·스마트키·이용안내·마이페이지), 예약·결제·대여·반납·연장 |
| 단지 관리자 | 고객·대여·반납검수·차량·매출·가동률 대시보드 |
| 최고관리자 | 전 단지·정산·쿠폰·시스템(점검모드·기능 차단) |

## Product Highlights

- **3역할:** 입주민 / 단지 관리자(staff) / 최고관리자
- **대여 유형:** 시간(최대 23h) · 일(29일) · 월(11개월)
- **결제:** 토스(카드·계좌이체·카카오페이), 빌링키 연장·면책금, 0원 결제(쿠폰+포인트)
- **운영:** 반납검수·면책금, 노쇼 자동 처리, 보험 만료 차단, RLS·Edge Function 방어
- **정책 엔진:** 취소·환불 구간(시간 1h / 일·월 72h·24h) 서버·앱 동기화

## Business Model (개요)

- 단지별 차량 공유·렌트 중개 (GT컴퍼니 운영, 플랫폼 수수료 구조)
- 최고관리자: 단지별 **플랫폼 수수료**(차량 1대당 월 과금 로직 구현)
- 정산: 단지별 월매출·정산요청·정산완료 워크플로

## Traction / 구현 완성도

- **200+** DB 마이그레이션, **30+** Edge Functions (결제·푸시·정산·스케줄)
- 입주민 온보딩 5단계, 면허 심사, 블랙리스트, FCM 시나리오 푸시
- 관리자 가동률(결제 예약 구간 기준), 타임라인·충돌위험, 실시간 대여 배너
- Android / iOS / Web(토스 리다이렉트) 지원

## Competitive Edge

1. **단지 스코프** — 입주민 인증·초대코드·RLS로 타 단지 데이터 격리
2. **정책 코드화** — 환불·요금 캡·노쇼·정산이 DB RPC·앱 FAQ 단일 출처
3. **운영 통합** — 예약~반납~검수~매출이 한 제품(별도 POS 불필요)
